# 日报/周报语气统一：通俗但正式（主人 2026-07-15 定稿）

## 背景

主人审阅 07-15 日报时先后两次纠正：第一版术语堆砌（开发才懂）→ 太技术；改口语版又「太通俗、不像日报」。最终定稿要求：**通俗但正式**——非技术主管能看懂，同时保持书面日报语气。此规则对日报、周报统一生效，后续都这么写。

## 坑 / 错误做法

- **过度技术**：如「sid宽表原地加列 is_valid/五档/source_type + 单张 dws_session_duration_d」——读者（部门/主管）看不懂。
- **过度口语**：如「查到根子上是他们把开关报成了0（关掉）」「来回聊了三轮」「没写死」「搞定了」——不符合日报的正式书面语气。

## 正确做法

写日报/周报正文时按「通俗但正式」：

1. 术语翻业务说法：
   - `dws_session_duration_d` → 「停留时长汇总表」
   - `attribution_flag=0` → 「归因标识为 0（未开启）」
   - `is_valid / duration_bucket` → 「有效标识 / 时长档位」
   - `ETL` → 「数据加工」；`DWS/DWM` → 「汇总层/中间层」
2. **通俗 ≠ 口语**：用书面词替口语——
   - 「根子上」→「根因」；「敲定/搞定」→「定稿/已完成」；「聊了三轮」→「经三轮联评」；「没写死」→「暂未固化 / 可配置」；「暂缓」「收敛」「定位」优先。
3. 禁「术语罗列式」：如「原地加列 is_valid/五档/source_type」→「会话表新增『有效标识、时长档位、来源』等字段」。
3b. **禁区间/代码记号直出**（bus 聊天黑话，主人 07-15 二次纠正）：`[5,60)`→「5 秒~1 分钟」；`(1800,43200]`→「30 分钟~12 小时」；`col_list/PK/dt 分区`→「字段清单/主键/按天分区」；表名、海豚宏（`$[yyyyMMdd-1]`）不进日报正文。
4. 每条约 25~45 字（以说清为准，不为凑字数砍关键信息）；`【今日结果】` 通常 3 条。
5. 仍守原铁律：只写数据工作、禁 bus#、写任务名、机器人/TG/环境运维不进正文。

定稿样例（07-15 日报「今日结果」首条）：
> 定位鼎丰(SF-98)无归因结果根因：客户端上报的归因标识为 0（未开启），非数据加工问题，已完成。
