# agent-bus「静默吞单」两坑：seal 误封 + needs_reply 漏判

## 背景

用户侧 TG 能看到狂人原文（mirror 正常），但又初长时间无 ACK/reply，体感「又没动静」。两次独立根因，症状相似，需用状态文件区分。

## 坑 A · `seal_backlog_processed` 误封（bus#944）

### 现象

- `processed.json` 有 `#944`，`sent.json` **无** `ack:bus:944` / `reply:bus:944`
- offset 跳到更新 bus 后，`bid < max_id` 的历史派单被一律 `mark_processed`

### 错误做法

```python
# 旧：凡 bid < max_id 全封存
mark_processed(agent, bid)
```

### 正确做法

封存时跳过「仍需 reply 且未结案」的入站：

```python
if message_needs_reply(text, from_agent=..., to_agent=...) and not has_reply(agent, bid):
    continue
mark_processed(agent, bid)
```

## 坑 B · `message_needs_reply` 漏判（bus#980）

### 现象

- 狂人直派 `to_agent=youchu_ai`，正文含「你先把 test 发版准备好」「git 先别 push」
- poller 判定 `needs_reply=False` → 当场 `mark_processed`，**不 wake、不 ACK**
- TG 有 incoming 事件，但 Cursor 主会话从未被唤醒

### 错误做法

仅靠窄触发词（派活/报我/在线吗…）；跟进式派工指令未覆盖即当「不用回」。

### 正确做法（2026-07-02 起）

1. **铁律**：`from_agent in (worker_ant, worker_ant_bot)` 且 `to_agent == youchu_ai` → **默认需 ack+reply**，除非正文 opt-out（不用回/learn_only/先待命…）
2. **补充触发词**：`你先把|先别 push|等我审|gate|diff|发版|终审`
3. `poll_once` / `health` / `seal_backlog` 传 `from_agent` + `to_agent` 给 `message_needs_reply()`

## 自查清单（用户说「没动静」时 60 秒内跑）

```bash
AGENT_BUS_STATE_DIR="$HOME/Library/Application Support/youchu-agent-bus/state" \
  bash .cursor/scripts/agent-bus-poller-check.sh --json

# 对具体 bus_id N：
rg '"N"' "$HOME/Library/Application Support/youchu-agent-bus/state/youchu_ai_"*.json*
rg 'ack:bus:N|reply:bus:N' "$HOME/Library/Application Support/youchu-agent-bus/state/youchu_ai_sent.json"
```

| 状态组合 | 诊断 |
|----------|------|
| processed ✓, ack/reply ✗ | 吞单（A 或 B）→ 补 ACK+reply + 修代码 |
| ack ✓, reply ✗ | 干到一半或 wake 失败 → 补 reply |
| pending=0, unreplied=[], 但用户仍等 | 多半已漏判 processed → 查 `message_needs_reply` |
| wake_feed 无对应 bus | poller 未 emit_wake 或 wake-bridge 死 |

## 坑 C · `to_agent=all` 上线广播漏 ack（bus#990）

### 现象

- 正文 `【worker_ant 上线】…可以开始交付`，`to_agent=all`
- 无「不用回」→ 用户期望短 ack，但 `needs_reply=False`（直派规则只认 `youchu_ai`）
- TG 显示「处理中」后无下文

### 正确做法

- 含 `worker_ant 上线` 且无 opt-out → `needs_reply=True`（短 ack 在线即可）
- 或匹配 `可以开始交付|通道恢复`
